## 1. 基础概念 (Basic Concepts)

- 数据压缩 (Data Compression) — 用比原始表示更少的比特来编码信息的过程,旨在节省存储空间和传输带宽 — https://en.wikipedia.org/wiki/Data_compression
- 无损压缩 (Lossless Compression) — 可完全重建原始数据的压缩技术,无任何信息损失,适用于文本和可执行文件 — https://en.wikipedia.org/wiki/Lossless_compression
- 有损压缩 (Lossy Compression) — 允许在重建过程中丢失部分非关键视觉或听觉信息以换取极高压缩比的技术 — https://en.wikipedia.org/wiki/Lossy_compression
- 码率 (Bit Rate) — 在数字多媒体中,单位时间内传输或处理的比特数,直接影响音视频质量与带宽消耗 — https://en.wikipedia.org/wiki/Bit_rate
- 压缩比 (Compression Ratio) — 未压缩数据大小与压缩后数据大小的比值,是衡量空间节省效率的直观指标 — https://en.wikipedia.org/wiki/Data_compression_ratio
- 数据冗余 (Data Redundancy) — 数据中存在的超出表达必要信息的多余或重复信号,是压缩技术消除的核心目标 — https://en.wikipedia.org/wiki/Data_redundancy
- 空间冗余 (Spatial Redundancy) — 图像或单帧信号中相邻像素或样本之间存在的高度物理特征相关性 — https://en.wikipedia.org/wiki/Intra-frame_coding
- 时间冗余 (Temporal Redundancy) — 视频序列中相邻帧之间存在的背景静止或物体平移带来的高度相似性 — https://en.wikipedia.org/wiki/Inter_frame
- 感知冗余 (Perceptual Redundancy) — 人类视觉或听觉系统(HVS/HAS)不敏感的物理信号细节,可通过掩蔽效应去除 — https://en.wikipedia.org/wiki/Psychoacoustics
- 统计冗余 (Statistical Redundancy) — 数据源中符号出现概率分布偏离均匀分布所导致的信息编码冗余 — https://en.wikipedia.org/wiki/Entropy_(information_theory)
- 信源编码 (Source Coding) — 泛指在数据存储或传输前,旨在最小化数据表示所需比特数的数学与工程过程 — https://en.wikipedia.org/wiki/Source_coding_theorem
- 字典 (Dictionary) — 压缩算法(如LZ77系列)中用于存储已见字符串或模式以实现短引用指针替换的核心数据结构 — https://en.wikipedia.org/wiki/Dictionary_coder
- 上下文 (Context) — 预测当前或未来符号时所依赖的先验信息、局部相邻像素或历史符号序列 — https://en.wikipedia.org/wiki/Context_mixing
- 概率模型 (Probability Model) — 描述数据源中各个符号在特定上下文中出现可能性的数学分布模型 — https://en.wikipedia.org/wiki/Statistical_model
- 码字 (Codeword) — 在编码字典或熵编码树中,代表特定信源符号或符号序列的二进制位串 — https://en.wikipedia.org/wiki/Code_word
- 编解码器 (Codec) — 能够对数字数据流或信号进行编码(压缩)和解码(解压)的软硬件系统集合 — https://en.wikipedia.org/wiki/Codec
- 数据差分 (Data Differencing/Delta) — 仅记录和传输源数据与目标数据之间增量差异的压缩特例 — https://en.wikipedia.org/wiki/Delta_encoding
- 率失真优化 (Rate-Distortion Optimization, RDO) — 在视频或图像编码中,在给定可用码率下通过拉格朗日乘数最小化失真程度的决策机制 — https://en.wikipedia.org/wiki/Rate%E2%80%93distortion_optimization
- 压缩伪影 (Compression Artifact) — 采用有损压缩(如高频量化截断)解码后在媒体中出现的块效应、振铃等视觉瑕疵 — https://en.wikipedia.org/wiki/Compression_artifact
- 有效载荷 (Payload) — 在数据传输分组或压缩数据容器中,除去协议头部和描述元数据后的实际核心压缩位流 — https://en.wikipedia.org/wiki/Payload_(computing)

## 2. 信息论 (Information Theory)

- 信息论 (Information Theory) — 研究信息的量化、存储和通信理论极限的数学理论分支,由香农创立 — https://en.wikipedia.org/wiki/Information_theory
- 信息熵 (Shannon Entropy) — 随机变量所包含信息量的期望值,代表无损数据压缩的理论比特下界 — https://en.wikipedia.org/wiki/Entropy_(information_theory)
- Kullback-Leibler散度 (KL Divergence) — 衡量使用近似概率分布代替真实概率分布进行编码时所产生的预期额外信息量(非对称) — https://en.wikipedia.org/wiki/Kullback%E2%80%93Leibler_divergence
- Jensen-Shannon散度 (JS Divergence) — 基于KL散度构建的平滑、对称且有界的概率分布相似度衡量指标,常用于深度学习 — https://en.wikipedia.org/wiki/Jensen%E2%80%93Shannon_divergence
- 互信息 (Mutual Information) — 衡量两个随机变量之间相互依赖程度的量,反映已知一个变量后另一个变量不确定性的减少量 — https://en.wikipedia.org/wiki/Mutual_information
- 率失真理论 (Rate-Distortion Theory) — 规定在允许一定均方误差或客观失真条件下,信源所需最小传输码率的理论界限 — https://en.wikipedia.org/wiki/Rate%E2%80%93distortion_theory
- 信道容量 (Channel Capacity) — 给定通信信道在渐近零错误率下能够可靠传输信息的最大理论速率 — https://en.wikipedia.org/wiki/Channel_capacity
- 马尔可夫链 (Markov Chain) — 具备无后效性的随机过程,即系统未来状态的概率仅依赖于当前状态而与过去历史无关 — https://en.wikipedia.org/wiki/Markov_chain
- 平稳过程 (Stationary Process) — 其无条件联合概率分布不随时间推移或时间平移而发生改变的随机过程 — https://en.wikipedia.org/wiki/Stationary_process
- 遍历性 (Ergodicity) — 随机过程的时间平均在概率上等于其状态空间(系综)平均的重要统计特性 — https://en.wikipedia.org/wiki/Ergodic_process
- 前缀码 (Prefix Code) — 任何有效码字都不是集合中其他任何码字前缀的编码系统,确保了码流的即时且无歧义解码 — https://en.wikipedia.org/wiki/Prefix_code
- 唯一可译码 (Uniquely Decodable Code) — 任何串联的码字序列只能被解析为唯一一种有效源符号序列的变长编码 — https://en.wikipedia.org/wiki/Variable-length_code
- 克拉夫特不等式 (Kraft-McMillan Inequality) — 判定给定一组码字长度集合是否存在对应的唯一可译码或前缀码的必要数学条件 — https://en.wikipedia.org/wiki/Kraft%E2%80%93McMillan_inequality
- 渐近等分性 (Asymptotic Equipartition Property, AEP) — 表明极长随机序列大多属于具有接近信源熵的"典型集"的信息论核心属性 — https://en.wikipedia.org/wiki/Asymptotic_equipartition_property
- 香农信源编码定理 (Shannon's Source Coding Theorem) — 确立了独立同分布数据压缩极限为数据源信息熵的经典定理 — https://en.wikipedia.org/wiki/Shannon%27s_source_coding_theorem
- 联合熵 (Joint Entropy) — 衡量包含多个随机变量的联合系统的总不确定性,必定小于等于各边缘熵之和 — https://en.wikipedia.org/wiki/Joint_entropy
- 条件熵 (Conditional Entropy) — 在已知一个随机变量的情况下,另一个随机变量剩余的不确定性,用于评估上下文模型的收益 — https://en.wikipedia.org/wiki/Conditional_entropy
- 交叉熵 (Cross Entropy) — 基于真实分布和非真实的预测分布对符号进行编码时的平均预期位长 — https://en.wikipedia.org/wiki/Cross_entropy
- Slepian-Wolf定理 (Slepian-Wolf Coding) — 定义了多个相关信源在独立编码、联合解码条件下的无损压缩理论界限(即总速率等于联合熵) — https://en.wikipedia.org/wiki/Distributed_source_coding
- Wyner-Ziv定理 (Wyner-Ziv Coding) — 带有解码器端边信息的有损信源编码定理,是分布式视频编码(DVC)的理论基石 — https://en.wikipedia.org/wiki/Distributed_source_coding

## 3. 熵编码 (Entropy Coding)

- 熵编码 (Entropy Encoding) — 根据符号出现的统计概率为其分配可变长度或分数位长的无损压缩底层方案 — https://en.wikipedia.org/wiki/Entropy_encoding
- 哈夫曼编码 (Huffman Coding) — 使用贪心算法二叉树构建最优前缀码的经典熵编码,但受限于单符号整数位长 — https://en.wikipedia.org/wiki/Huffman_coding
- 香农-法诺编码 (Shannon-Fano Coding) — 基于概率递减排序并将集合递归二分的早期前缀编码方法,效率略逊于哈夫曼 — https://en.wikipedia.org/wiki/Shannon%E2%80%93Fano_coding
- 算术编码 (Arithmetic Coding) — 将整个输入消息编码为介于0和1之间的一个高精度分数的变长熵编码,压缩率极高 — https://en.wikipedia.org/wiki/Arithmetic_coding
- 范围编码 (Range Coding) — 算术编码的一种变体,通常以字节或大基数而非比特为单位进行重归一化,速度更快 — https://en.wikipedia.org/wiki/Range_coding
- 非对称数字系统 (Asymmetric Numeral Systems, ANS) — 由Jarek Duda发明,结合了算术编码的压缩率和哈夫曼查表速度的现代熵编码族 — https://en.wikipedia.org/wiki/Asymmetric_numeral_systems
- 表格ANS (tANS) — ANS的一种基于状态机的查表变体,在不需要乘法的情况下提供极高的解码流水线吞吐量 — https://en.wikipedia.org/wiki/Asymmetric_numeral_systems
- 范围ANS (rANS) — 适用于大字母表的ANS变体,利用算术公式替代全表,广泛应用于JPEG XL等现代媒体标准 — https://en.wikipedia.org/wiki/Asymmetric_numeral_systems
- 均匀非对称二元系统 (uABS) — ANS家族中最基础的二元形态,专门处理仅包含0和1且概率非均匀分布的比特流 — [URL待查]
- 哥伦布编码 (Golomb Coding) — 为遵循几何分布的无限输入数据(如较小的非负整数偏多)设计的一种极简且高效的熵编码 — https://en.wikipedia.org/wiki/Golomb_coding
- 莱斯编码 (Rice Coding) — 哥伦布编码的特例,其除数参数限制为2的幂,从而允许使用极速的位移操作替代除法运算 — https://en.wikipedia.org/wiki/Golomb_coding
- 通斯托尔编码 (Tunstall Coding) — 将可变长度的源符号序列映射到恒定长度码字的独特熵编码,对单比特错误具有弹性 — https://en.wikipedia.org/wiki/Tunstall_coding
- 指数哥伦布编码 (Exponential-Golomb Coding) — 一种参数化的通用变长编码,广泛用于H.264/HEVC视频标准中以编码大小变动剧烈的语法元素 — https://en.wikipedia.org/wiki/Exponential-Golomb_coding
- 上下文混合 (Context Mixing) — 结合多个独立统计模型(如通过神经网络)的预测以生成极高准确率概率估计的算法技术(如PAQ) — https://en.wikipedia.org/wiki/Context_mixing
- 逻辑混合 (Logistic Mixing) — 在PAQ算法族中,通过对数几率(Logistic)域合并概率预测以赋予确定性极高的极端预测更大权重的机制 — https://en.wikipedia.org/wiki/Context_mixing
- 预测部分匹配 (Prediction by Partial Matching, PPM) — 基于定长历史上下文(N-gram)自适应预测下一个符号出现概率的高压缩率统计建模技术 — https://en.wikipedia.org/wiki/Prediction_by_partial_matching
- 游程编码 (Run-Length Encoding, RLE) — 将连续重复的相同数据序列替换为单个数据值及其重复计数的最基础熵压缩形式 — https://en.wikipedia.org/wiki/Run-length_encoding
- 埃利亚斯伽马编码 (Elias Gamma Coding) — 针对正整数设计的通用编码,常用于数值大小不可预知但倾向于小数值的先验分布 — https://en.wikipedia.org/wiki/Elias_gamma_coding
- 斐波那契编码 (Fibonacci Coding) — 基于斐波那契数列表示整数的变长前缀码,利用连续两个1作为码字结束标识,天然具备错误恢复同步特性 — https://en.wikipedia.org/wiki/Fibonacci_coding
- 一元编码 (Unary Coding) — 用n个1后接一个0(或反之)来表示自然数n的极简通用前缀码,常作为其他复杂编码的组成部分 — https://en.wikipedia.org/wiki/Unary_coding

## 4. 字典编码 (Dictionary Coding)

- 字典编码 (Dictionary Coding) — 通过在文本中搜索匹配子串并将其替换为指向较早出现位置的引用元组的无损压缩类 — https://en.wikipedia.org/wiki/Dictionary_coder
- LZ77 — 1977年提出的使用滑动窗口寻找匹配并输出(距离,长度,下一字符)的基石级字典算法 — https://en.wikipedia.org/wiki/LZ77_and_LZ78
- LZ78 — 放弃滑动窗口,通过动态构建显式短语树词典进行逐符号扩展的压缩算法 — https://en.wikipedia.org/wiki/LZ77_and_LZ78
- LZW (Lempel-Ziv-Welch) — LZ78的改进版,预置全字母单字符字典,曾因Unisys对其申请软件专利引发著名的GIF争议 — https://en.wikipedia.org/wiki/Lempel%E2%80%93Ziv%E2%80%93Welch
- DEFLATE — 结合了LZ77滑动窗口算法与哈夫曼编码的混合压缩标准,广泛定型于ZIP档案和Gzip工具中 — https://en.wikipedia.org/wiki/Deflate
- LZMA (Lempel-Ziv-Markov chain algorithm) — 使用带有马尔可夫模型的大规模字典和算术/范围编码的高压缩比算法,常用于7z格式 — https://en.wikipedia.org/wiki/Lempel%E2%80%93Ziv%E2%80%93Markov_chain_algorithm
- Zstandard (zstd) — 由Yann Collet为Facebook开发的现代压缩算法,兼顾类似LZMA的高压缩比与近乎LZ4的极高解压速度 — https://en.wikipedia.org/wiki/Zstd
- Snappy — Google开发的针对MapReduce等大数据平台RPC系统优化的无损压缩库,不使用熵编码,追求极致速度 — https://en.wikipedia.org/wiki/Snappy_(compression)
- Brotli — Google开发的包含120KB内置静态Web词典和二阶上下文建模的算法,广泛成为下一代HTTP/HTTPS流压缩标准 — https://en.wikipedia.org/wiki/Brotli
- LZ4 — 提供极快压缩与解压速度(多核环境下达数GB/s)的面向字节的LZ77变体,广泛用于文件系统级透明压缩 — https://en.wikipedia.org/wiki/LZ4_(compression_algorithm)
- LZO (Lempel-Ziv-Oberhumer) — 注重极致解压缩速度和极低内存占用的无损数据压缩库,尤其适用于实时嵌入式环境 — https://en.wikipedia.org/wiki/Lempel%E2%80%93Ziv%E2%80%93Oberhumer
- 滑动窗口 (Sliding Window) — LZ77类型算法中用来缓存最近处理过的数据序列以进行反向字符串匹配定界的内存缓冲区 — https://en.wikipedia.org/wiki/Sliding_window_protocol
- 哈希链 (Hash Chain) — 字典编码器内部用于快速定位相同前缀子串历史位置的散列表串联链接结构,极大加速了模式搜索 — [URL待查]
- 延迟匹配 (Lazy Match) — Deflate等算法中的一种优化启发式策略:当发现当前匹配后,推迟输出并检查下一个字节是否能产生更长匹配 — https://en.wikipedia.org/wiki/Deflate
- 最优解析 (Optimal Parsing) — 将输入字符串全局解析为能使最终编码比特流最小化的匹配序列(通常通过图最短路径算法实现) — [URL待查]
- 字典训练 (Dictionary Training) — Zstd等现代压缩器提供的功能,通过分析特定结构的小型数据集语料预先生成定制化热点字典以提升小文件压缩率 — https://en.wikipedia.org/wiki/Zstd
- 匹配长度与距离 (Length-Distance Pair) — 字典编码器向解码器发送的核心描述元组,指示从滑动窗口的相对后退步数和复制的字节数 — https://en.wikipedia.org/wiki/LZ77_and_LZ78
- Smaz — 专为压缩非常短的字符串(如URL或小型数据库文本字段)而设计的微型启发式静态字典压缩库 — https://en.wikipedia.org/wiki/Smaz
- Bzip2 — 严格意义上虽不是字典编码,但常与同类工具比较;基于Burrows-Wheeler变换和MTF,无滑动窗口限制且压缩率优于Gzip — https://en.wikipedia.org/wiki/Bzip2
- 静态字典 (Static Dictionary) — 预先硬编码在压缩器和解压器中的常量字典(如Brotli的HTML/单词库),对于处理未累积足够上下文的数据首部至关重要 — https://en.wikipedia.org/wiki/Brotli

## 5. 变换 (Transform)

- 变换编码 (Transform Coding) — 将数据从空间域或时间域映射到频域以便于集中能量、隔离高频细节进行量化和压缩的技术 — https://en.wikipedia.org/wiki/Transform_coding
- Burrows-Wheeler变换 (BWT) — 通过对输入字符串的所有轮转进行排序,将拥有相同上下文的字符聚拢的无损块级变换重排算法 — https://en.wikipedia.org/wiki/Burrows%E2%80%93Wheeler_transform
- 前移变换 (Move-to-Front, MTF) — 将最近出现过的字符移到列表前端,从而把BWT产生的连续局部字符簇转换为小整数流的编码步骤 — https://en.wikipedia.org/wiki/Move-to-front_transform
- 游程编码 (Run-Length Encoding, RLE) — 将连续重复的符号序列替换为该符号及其出现次数的一种极其基础但高效的压缩形式 — https://en.wikipedia.org/wiki/Run-length_encoding
- 离散余弦变换 (Discrete Cosine Transform, DCT) — 使用不同频率的余弦函数和表示有限信号序列,是JPEG、MPEG等绝大多数多媒体有损压缩的数学基础 — https://en.wikipedia.org/wiki/Discrete_cosine_transform
- 改进离散余弦变换 (Modified Discrete Cosine Transform, MDCT) — 在连续处理块间应用交叠操作以消除边界伪影的重叠变换,广泛用于MP3和AAC音频压缩 — https://en.wikipedia.org/wiki/Modified_discrete_cosine_transform
- 小波变换 (Wavelet Transform) — 利用有限长、会衰减的小波基函数对信号进行多分辨率时频分析,是JPEG 2000彻底消除块效应的核心 — https://en.wikipedia.org/wiki/Wavelet_transform
- 量化 (Quantization) — 将连续或大范围的高精度离散值映射到一组较小的代表性离散值(通常引入不可逆截断)的有损压缩步骤 — https://en.wikipedia.org/wiki/Quantization_(image_processing)
- 量化矩阵 (Quantization Matrix) — 预设的除数系数表格,用于DCT后依人类视觉生理特性对不同频率分量施加不同的压缩力度(高频强压) — https://en.wikipedia.org/wiki/Quantization_(image_processing)
- 差分脉冲编码调制 (Differential Pulse-code Modulation, DPCM) — 基于预测模型,仅量化和编码当前样本与前一个样本预测值之间极小差值的信号处理技术 — https://en.wikipedia.org/wiki/Differential_pulse-code_modulation
- 增量编码 (Delta Encoding) — 存储文件或流式数据集之间差值(Delta)的通用数据缩减技术,极大地减小了版本更新的体积 — https://en.wikipedia.org/wiki/Delta_encoding
- 线性预测编码 (Linear Predictive Coding, LPC) — 基于语音声带模型等历史信号的线性组合来估计未来值的音频压缩技术,常用于VoIP — https://en.wikipedia.org/wiki/Linear_predictive_coding
- 色度二次采样 (Chroma Subsampling) — 降低颜色通道分辨率而保持亮度通道完整的做法(如4:2:0),利用人眼对色彩细节敏感度低的特性节省带宽 — https://en.wikipedia.org/wiki/Chroma_subsampling
- 子带编码 (Subband Coding) — 将宽带信号滤波分解为多个不同频带,并根据各频带的信息量独立分配位进行编码的技术 — https://en.wikipedia.org/wiki/Sub-band_coding
- 帧内预测 (Intra-frame Prediction) — 在单一视频帧或图片内部,利用边缘已编码像素向未编码宏块进行纹理外推预测以消除空间冗余 — https://en.wikipedia.org/wiki/Intra-frame_coding
- 帧间预测 (Inter-frame Prediction) — 视频编码中通过引用时间线上前后邻近的其他帧(如P帧、B帧)来预测当前帧,消除时间冗余的机制 — https://en.wikipedia.org/wiki/Inter_frame
- 运动估计 (Motion Estimation) — 视频编码中在相邻帧间寻找最佳匹配块,并生成描述两帧相对位移的运动向量(Motion Vector)的过程 — https://en.wikipedia.org/wiki/Motion_estimation
- 运动补偿 (Motion Compensation) — 解码器根据运动向量从参考帧中提取像素块,并与传输的残差相加以重构当前视频帧的技术 — https://en.wikipedia.org/wiki/Motion_compensation
- 条带与瓦片 (Strip & Tile) — 将大型图像或视频帧分割为若干矩形或长条形独立可解码区域以支持多线程并行变换与容错的结构 — https://en.wikipedia.org/wiki/JPEG_XL
- 宏块 (Macroblock) — 传统基于块的运动补偿视频压缩格式(如H.264)中用作色彩变换与预测基本处理单元的16x16像素区域 — https://en.wikipedia.org/wiki/Macroblock

## 6. 学习型压缩 (Learned Compression)

- 变分自编码器 (Variational Autoencoder, VAE) — 一种生成式概率模型,通过编码器将数据压缩为隐空间的高斯分布参数,再由解码器重构,是学习型图像压缩的骨干基石 — https://en.wikipedia.org/wiki/Variational_autoencoder
- 标准化流 (Normalizing Flow) — 运用一系列可逆且计算行列式极其高效的变量代换变换,将简单先验分布映射为复杂数据分布的生成式架构 — https://en.wikipedia.org/wiki/Flow-based_generative_model
- 连续标准化流 (Continuous Normalizing Flow, CNF) — 通过神经常微分方程(Neural ODEs)定义连续时间矢量场,实现隐空间分布间平滑演化的无损可逆流模型 — https://en.wikipedia.org/wiki/Flow-based_generative_model
- 扩散模型 (Diffusion Model) — 通过学习反转马尔可夫渐进式加噪过程,在极低压缩码率下能根据文本或稀疏边缘重构或"幻觉"出高保真纹理细节的生成模型 — https://en.wikipedia.org/wiki/Diffusion_model
- 生成对抗网络 (Generative Adversarial Network, GAN) — 由生成器和判别器相互博弈构成的模型,常用于损失函数中以补偿传统MSE指标导致的模糊,生成感知上极度逼真的细节 — https://en.wikipedia.org/wiki/Generative_adversarial_network
- 变换器模型 (Transformer Model) — 采用注意力机制捕捉全局依赖的深度学习架构,近年来在神经视频和大规模时间序列压缩中用于建立长程上下文关联 — https://en.wikipedia.org/wiki/Transformer_(machine_learning_model)
- 超先验 (Hyperprior) — 在贝叶斯统计中作用于先验参数之上的分布;在神经压缩中作为额外编码的极小规模侧信息,用于向解码器传递隐表示的空间结构分布特征 — https://en.wikipedia.org/wiki/Hyperprior
- 尺度超先验 (Scale Hyperprior) — 超先验的演进结构,专注于动态预测隐空间特征图在不同空间位置的方差(尺度),从而使熵编码器精确适配局部信号活跃度 — [URL待查]
- 自回归模型 (Autoregressive Model) — 在解码隐特征时,利用已经解码的相邻元素预测当前元素的统计模型,与超先验联合可进一步挤压空间冗余 — https://en.wikipedia.org/wiki/Autoregressive_model
- 熵瓶颈 (Entropy Bottleneck) — 神经压缩架构中的咽喉模块,它模拟非可导的舍入量化过程(通常在训练期加均匀噪声),并估计隐表示的概率质量函数(PMF)以计算比特率 — [URL待查]
- 神经率失真优化 (Neural Rate-Distortion Optimization) — 在端到端网络训练的损失函数中显式集成可微的比特率估计项(Rate)和失真项(Distortion)进行联合拉格朗日优化的机制 — https://en.wikipedia.org/wiki/Rate%E2%80%93distortion_optimization
- 隐式神经表示 (Implicit Neural Representation, INR) — 将图像或3D场景直接参数化为从坐标(x,y)映射到像素值(RGB)的多层感知机(MLP)权重,实现了真正的无分辨率限制压缩 — https://en.wikipedia.org/wiki/Neural_field
- iVPF / iFlow — 特指在视频压缩和插帧中,使用隐式矢量场(Implicit Vector Field)或光流场作为变形依据的神经编解码网络组件 — [URL待查]
- L3C (Learned Low-level Luminescence Compression) — 专注于低光或HDR等极端统计分布环境下的专门优化的学习型底层图像压缩范式 — [URL待查]
- 通道条件建模 (Channel-conditional Modeling) — 神经压缩中为解决自回归模型由于串行解码过慢的缺陷,将隐特征按通道分组,利用已编码的通道预测当前通道以实现高并发吞吐的技术 — [URL待查]
- 模型压缩 (Model Compression) — 致力于减小经过训练的庞大深度神经网络参数规模(如LLM权重)以降低推理内存、提升吞吐的技术集合 — https://en.wikipedia.org/wiki/Model_compression
- 训练后量化 (Post-Training Quantization, PTQ) — 在神经网络训练完成后,通过标定和截断将浮点权重直接映射为低精度整数(如INT4)而无需巨额算力重训的压缩方法 — [URL待查]
- 量化感知训练 (Quantization-Aware Training, QAT) — 在网络训练的前向传播过程中插入伪量化节点以模拟误差,使模型参数主动适应低精度离散环境的技术 — [URL待查]
- 残差向量量化 (Residual Vector Quantization, RVQ) — 神经音频压缩(如EnCodec)中,通过多级码本级联(下一级仅量化上一级重建后的残差),实现流式极低码率的高效表征法 — [URL待查]
- 感知损失 (Perceptual Loss) — 不采用像素级的绝对差异(L1/L2),而是比较两幅图像在预训练CNN分类器(如VGG)提取的高层语义特征图上的欧氏距离,极大提升了压缩解压后图像的结构保真度 — [URL待查]

## 7a. 评价指标 - 质量 (Quality)

- 峰值信噪比 (Peak Signal-to-Noise Ratio, PSNR) — 基于均方误差对数缩放的客观指标,衡量失真信号相对原始最大信号强度的数学误差水平,但常与人眼感知脱节 — https://en.wikipedia.org/wiki/Peak_signal-to-noise_ratio
- 结构相似度 (Structural Similarity, SSIM) — 综合比较两幅图像在亮度、对比度和局部结构方差的指标,比PSNR更贴近人类视觉的主观感知 — https://en.wikipedia.org/wiki/Structural_similarity
- 多尺度结构相似度 (MS-SSIM) — 在多个不同下采样分辨率尺度上执行结构相似度评估,以兼顾宏观结构和微观细节的改进版SSIM — https://en.wikipedia.org/wiki/Structural_similarity
- 学习感知图像块相似度 (Learned Perceptual Image Patch Similarity, LPIPS) — 通过提取参考图与失真图在预训练分类网络(如VGG)的深层激活映射差异来评估视觉距离的感知标尺 — [URL待查]
- Fréchet Inception距离 (Fréchet Inception Distance, FID) — 对比真实图像和解压生成图像在Inception网络中深层特征激活的均值与协方差分布距离的生成模型评价金准则 — https://en.wikipedia.org/wiki/Fr%C3%A9chet_inception_distance
- 视频多方法评估融合 (Video Multimethod Assessment Fusion, VMAF) — Netflix联合研发的、通过支持向量机融合视觉信息保真度与细节损失等多种特征的视频客观质量预测指标 — https://en.wikipedia.org/wiki/Video_Multimethod_Assessment_Fusion
- VMAF-NEG (VMAF No Enhancement Gain) — VMAF的抗欺骗变体,专门限制并在特征层面剔除因对比度锐化等预处理伪图像增强操作带来的得分虚高 — [URL待查]
- 平均主观意见分 (Mean Opinion Score, MOS) — 评估压缩后视听多媒体质量的终极人本标准,由大量人类评估者在受控环境下主观打分后求算术平均获得 — https://en.wikipedia.org/wiki/Mean_opinion_score
- 深层图像结构与纹理相似度 (Deep Image Structure and Texture Similarity, DISTS) — 一种基于深度特征映射、具备抗微小空间位移偏差特性的鲁棒性纹理感知相似度测度指标 — [URL待查]
- Bjøntegaard Delta码率 (BD-Rate) — 在视频压缩标准化中不可或缺的工具,通过对两条率失真曲线拟合多项式的积分区间面积,量化新编解码器节省的绝对码率百分比 — https://en.wikipedia.org/wiki/Bjontegaard_metric
- 视觉信息保真度 (Visual Information Fidelity, VIF) — 建立在自然场景统计和人类视觉神经前馈模型基础上的度量,评估参考图像信息穿过失真信道后人脑提取量 — [URL待查]
- 感知损失 (Perceptual Loss) — 深度学习训练中替代逐像素L1/L2的损失函数,确保生成网络输出的图像与目标在高级语义结构上高度一致 — [URL待查]

## 7b. 评价指标 - 性能 (Performance)

- 压缩比 (Compression Ratio) — 衡量数据存储占用缩减效率的基石参数,等于未压缩原始数据字节数除以压缩后封包数据字节数 — https://en.wikipedia.org/wiki/Data_compression_ratio
- 压缩速度 (Compression Speed) — 编码器将原始载荷处理、变换并压缩封包写入的速率,常受字典搜索深度和模型复杂度制约,以 MB/s 表示 — [URL待查]
- 解压速度 (Decompression Speed) — 解码器还原出原始数据的吞吐速率,对于内容分发网络(CDN)前端响应及流媒体客户端缓冲体验极为关键 — [URL待查]
- 吞吐量 (Throughput) — 通信网络节点或系统总线上压缩数据流成功传递及处理的端到端并发实际速率,常直接受解压引擎带宽影响 — https://en.wikipedia.org/wiki/Network_throughput
- 内存占用 (Memory Footprint) — 编解码器在运行时执行滑动窗口匹配、上下文模型缓存或深层网络张量推理所需的峰值RAM存储开销 — https://en.wikipedia.org/wiki/Memory_footprint
- 延迟 (Latency) — 从输入第一个原始字节开始到接收端成功解压重构出相应数据块所需的总体时间跨度,在VoIP及云游戏中要求极严苛 — https://en.wikipedia.org/wiki/Latency_(engineering)
- 能耗 (Energy Consumption) — 算法在特定硅硬件架构上执行压缩/解压周期所导致的耗电瓦时数,决定了移动端视频播放的电池续航命脉 — https://en.wikipedia.org/wiki/Energy_consumption
- CPU利用率 (CPU Utilization) — 压缩算法执行期间占用中央处理器资源(不含I/O等待)的百分比负荷,关系到操作系统并行调度能力 — https://en.wikipedia.org/wiki/CPU_time
- GPU加速 (GPU Acceleration) — 利用图形处理器的海量并行处理单元(如CUDA或OpenCL)卸载块匹配、DCT变换或Brotli-G等SIMT解压计算的优化技术 — https://en.wikipedia.org/wiki/Hardware_acceleration
- SIMD友好度 (SIMD Friendliness) — 算法底层算子和循环结构对单指令多数据(如AVX-512、ARM Neon)并行向量化指令集架构的自动展开适配程度 — https://en.wikipedia.org/wiki/Single_instruction,_multiple_data
- Silesia语料库 (Silesia corpus) — 2003年建立的著名无损基准测试语料库,涵盖了数据库、医学影像切片、系统二进制文件等现代计算环境下的真实文件 — [URL待查]
- Canterbury语料库 (Canterbury corpus) — 1997年由坎特伯雷大学建立的用于取代古典卡尔加里语料库的标准化无损测试基准,注重文件类型的代表性平均 — https://en.wikipedia.org/wiki/Canterbury_corpus
- Enwik9基准测试 (Enwik9 benchmark) — 包含10^9字节(1GB)英文维基百科XML原始文本的巨大测试数据集,重点测试长程冗余检测与语义模型构建 — [URL待查]
- Hutter奖 (Hutter Prize) — 对成功缩小enwik9联合压缩文件加解压器自身体积的开发者发放巨额奖金的公开赛事,以测试AI算法提取人类知识信息熵的深度 — https://en.wikipedia.org/wiki/Hutter_Prize

## 8. 工业标准 (Industry Standards)

- JPEG — 基于8x8 DCT分块的经典有损图像压缩联合专家组标准,统治了早期数码摄影和Web图像三十年 — https://en.wikipedia.org/wiki/JPEG
- JPEG 2000 — 采用小波变换(Wavelet)替代DCT以消除高压块效应并原生支持多尺度分辨率解码的改进型医学及电影图像标准 — https://en.wikipedia.org/wiki/JPEG_2000
- JPEG XL — ISO/IEC 18181定义的下一代免版税图像格式,通过VarDCT和模块化模式提供卓越并发性能,且能直接数学无损地重构原JPEG文件 — https://en.wikipedia.org/wiki/JPEG_XL
- WebP — Google基于VP8视频编解码器I帧技术提取开发的支持有损、无损和Alpha透明度通道的网页图像格式 — https://en.wikipedia.org/wiki/WebP
- 高效率图像文件格式 (HEIF/HEIC) — 基于HEVC先进视频帧内预测及DCT技术构建的图像容器,在Apple生态中普及,以同等质量体积仅为JPEG一半著称 — https://en.wikipedia.org/wiki/High_Efficiency_Image_File_Format
- AVIF — 开放媒体联盟(AOMedia)利用AV1视频编码器的开源免版税、高保真(10/12-bit)特性打造的新锐图像和图像序列容器格式 — https://en.wikipedia.org/wiki/AVIF
- BPG (Better Portable Graphics) — 基于HEVC帧内编码打造的一款具有极高压缩率的优异图像格式,但因受限于HEVC复杂的专利池而未能成为网页通用标准 — https://en.wikipedia.org/wiki/Better_Portable_Graphics
- H.264 / AVC (Advanced Video Coding) — 占据长达十几年行业绝对统治地位、基于宏块化、四分之一像素精度的运动补偿视频压缩经典国际标准 — https://en.wikipedia.org/wiki/Advanced_Video_Coding
- H.265 / HEVC (High Efficiency Video Coding) — 引入最大64x64编码树单元(CTU)替换固定宏块,在同等4K画质下相较AVC节省50%码率的重磅视频标准 — https://en.wikipedia.org/wiki/High_Efficiency_Video_Coding
- H.266 / VVC (Versatile Video Coding) — JVET于2020年定稿的针对4K/8K/HDR/全景视频优化的极致压缩标准,引入高度复杂非正方形切分以获取最高压缩比 — https://en.wikipedia.org/wiki/Versatile_Video_Coding
- AV1 (AOMedia Video 1) — 开放媒体联盟应对专利战开发的专为互联网流媒体设计的开源、免版税的极具竞争力(匹敌HEVC/VVC)的视频编码格式 — https://en.wikipedia.org/wiki/AV1
- 基础视频编码 (EVC, MPEG-5 Part 1) — 提供基于20年以上已过期专利的完全免版税Baseline配置,并允许单独开关21种高级预测模块的分层视频标准 — https://en.wikipedia.org/wiki/Essential_Video_Coding
- 低复杂度增强视频编码 (LCEVC, MPEG-5 Part 2) — 采用解码低分辨率基础流(可用硬解)并在上层叠加低算力专属残差子层以重建高分辨率的革命性提升编解码效率的标准 — https://en.wikipedia.org/wiki/LCEVC
- MP3 (MPEG-1 Audio Layer III) — 奠定数字音乐时代的标志性有损音频压缩格式,利用人类心理声学模型遮蔽法则大量抹去听觉盲区的高频信息 — https://en.wikipedia.org/wiki/MP3
- 高级音频编码 (AAC) — 旨在作为MP3继任者的MPEG标准格式,使用更高精度的MDCT并在更低码率下提供更优质的高频保真,被Apple广泛采用 — https://en.wikipedia.org/wiki/Advanced_Audio_Coding
- Opus — IETF标准化的开放、免版税高度通用音频格式,巧妙融合了SILK和CELT引擎,无缝兼容极低延迟语音(VoIP)到无损质量音乐流播 — https://en.wikipedia.org/wiki/Opus_(audio_format)
- 免费无损音频编解码器 (FLAC) — 使用线性预测编码实现高采样率无损存储的开源数字音频格式,深受发烧友和数字音乐档案馆青睐 — https://en.wikipedia.org/wiki/FLAC
- 苹果无损音频编解码器 (ALAC) — Apple开发的类似于FLAC的位对位无损格式,用于iTunes高分辨率无损音乐库,并于2011年完全开源 — https://en.wikipedia.org/wiki/Apple_Lossless
- 增强型语音服务 (EVS) — 部署于第四代及第五代移动通信(VoLTE/VoNR)网络的超宽带音频编解码器,具备极高语音清晰度和抵抗网络抖动的信道容错性 — https://en.wikipedia.org/wiki/Enhanced_Voice_Services
- EnCodec — Meta(Facebook)推出的运用神经隐式模型与残差向量量化技术,以极端的极低码率对流媒体音频进行AI生成式重构的端到端神经网络音频编解码器 — https://en.wikipedia.org/wiki/EnCodec

## 9. 应用领域 (Application Domains)

- 图像压缩 (Image Compression) — 致力于消除二维光栅数组中的空间和统计冗余以最小化数字图像位元体积的算法应用工程 — https://en.wikipedia.org/wiki/Image_compression
- 视频压缩 (Video Compression) — 在单帧压缩之上叠加复杂的运动估计机制以削减时间维度的深层冗余,是流媒体平台的命脉技术 — https://en.wikipedia.org/wiki/Video_coding_format
- 音频编码 (Audio Coding) — 对数字声学波形(如音乐录音)进行高效编码和解构的技术,常高度依赖基于听觉掩蔽的心理声学滤波 — https://en.wikipedia.org/wiki/Audio_coding_format
- 语音编码 (Speech Coding) — 利用人体发声(声道共振峰、基频等)物理特征建模,专门针对人声对话、要求极低算法处理延迟的压缩通信应用 — https://en.wikipedia.org/wiki/Speech_coding
- 基因组压缩 (Genomic Data Compression) — 为高效归档、读取及跨机构传输DNA测序流(如FASTQ/BAM高维格式)而设计的超大基数无损压缩数学体系 — [URL待查]
- 点云压缩 (Point Cloud Compression) — 对激光雷达(LiDAR)或3D结构扫描产生的包含巨量不规则分布的三维(X,Y,Z)空间坐标及附加色彩的高维数据集实施的压缩方案 — [URL待查]
- 科学数据压缩 (Scientific Data Compression) — 专为气候模拟、高能物理及流体力学等HPC中心产生的大规模密集浮点数组而定制、极易利用精度自适应截断和并行计算的压缩前沿领域 — [URL待查]
- 列式数据库压缩 (Column-oriented DBMS Compression) — 数据仓库中利用同一列数据类型高度同质化(极易适用游程编码与字典映射)的特性进行极其激进的大块压缩以加速向量化查询的数据库应用 — https://en.wikipedia.org/wiki/Column-oriented_DBMS
- 日志数据压缩 (Log Data Compression) — 利用极深的历史滑动窗口及高度自适应的共享字典模式大幅剔除服务器群海量机器文本日志重复结构以节约存储的基础设施工程 — [URL待查]
- 时间序列压缩 (Time Series Compression) — 面向物联网(IoT)和服务器监控指标,利用浮点数的异或差值(XOR Delta)或二次差分机制(如著名Gorilla算法)在极小开销下进行的特征降维系统 — https://en.wikipedia.org/wiki/Time_series_database
- 神经网络压缩 (Neural Network Compression) — 涵盖结构剪枝、知识蒸馏和层级融合,旨在大幅削减将千亿参数深度模型部署至消费级设备所需RAM体积与计算浮点的核心AI工程体系 — https://en.wikipedia.org/wiki/Model_compression
- 模型量化 (Model Quantization) — 深度学习加速的核心应用,在降低神经网络权重张量运算浮点精度的同时(如FP16至INT4),通过极小化离群点影响使推理准确率下降最小化 — [URL待查]
- 低秩因式分解 (Low-rank Factorization) — 将大模型中庞大而冗余的全连接权重矩阵通过SVD等数学近似拆解为数个低秩小矩阵相乘,大幅节约内存加载带宽的技术分支 — [URL待查]
- 遥感卫星数据压缩 (Remote Sensing Data Compression) — 应对太空极高辐射环境及极窄下行数据链星地宽带限制的高光谱、多光谱成图硬件实时压缩应用(需极低运行功耗约束) — [URL待查]
- 高性能文件系统压缩 (HPC File System Compression) — 深入嵌入至现代操作系统底层(如ZFS, Btrfs内核模块),在文件块落盘时透明、零感知且毫秒级响应调用的块流式存储基石应用(常采用LZ4或zstd) — [URL待查]
- 可执行文件压缩 (Executable Compression) — 在不改变原本程序执行行为的前提下,将编译后的二进制文件(PE/ELF等)进行无损脱壳打包,用以节约磁盘并在安全攻防中加大静态逆向工程难度的安全与分发应用 — https://en.wikipedia.org/wiki/Executable_compression
- 纹理压缩 (Texture Compression) — 直接长期储存在GPU高显存带宽中,并允许着色器针对单点像素的渲染要求进行O(1)级别的随机抽样解压读取的特定分块有损图像压缩架构(如ASTC、DXT) — https://en.wikipedia.org/wiki/Texture_compression
