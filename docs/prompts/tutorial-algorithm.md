# 教程生成 — 算法原理模板

## 输入
- 教程 ID: `TUT-02xx` (P1,原理层)
- 算法名: 例如 "算术编码" / "LZMA" / "PAQ8px"
- 算法别名: 英文名 (例如 "Arithmetic Coding")
- 关联术语: 例如 `Arithmetic Coding`, `Range Coding`, `Entropy`
- 关联论文: 例如 `LIT-0005`, `LIT-0018`
- 关联资源: 例如 `Matt Mahoney's Block Sorting Compressor Test Page`
- 前置教程: 例如 `TUT-0101`, `TUT-0102`

## 结构 (7 段,比入门多 2 段)
1. **为什么读这个** — 一句话,这个算法能解决啥问题
2. **前置知识** — 术语 + 教程
3. **核心直觉** — 一段类比,先建立图景 (例如:算术编码 = 把消息编成 [0,1) 一个实数)
4. **形式化定义** — 数学公式 + 符号表 (但不超过 5 个公式)
5. **代码骨架** — 关键伪代码或 30 行真实代码
6. **优缺点对比** — 用 `<AlgorithmCompareTable>` 组件
7. **常见坑** — 至少 5 条
8. **下一步** — 链到下一篇原理或专题教程

## 字数: 1500-3000 字

## 必填字段
- `linkedTerms`: 至少 3 个
- `linkedPapers`: 至少 1 个
- `codeExamples`: 至少 1 个 (Python/C 都行,标注语言)
- `comparison`: 至少 1 个对比表

## 风格
- 先讲直觉,再上公式
- 公式不堆,每个公式紧跟一段白话翻译
- 代码必须能跑通 (除非是教学伪代码,要标注 `pseudo`)

## 联网核查
- 算法名 / 作者 / 年份 / 关键论文必须联网核实
- SOTA 数字必须从 `leaderboards.ts` 引
