# 教程生成 — 项目实操模板

## 输入
- 教程 ID: `TUT-01xx` (P0 实操类,例如 TUT-0107 Silesia 评测实操)
- 任务: 例如 "跑通 Silesia 评测"
- 关联资源: `projectAssets.json` 中的 ASSET-XXXX
- 关联脚本: `scripts/` 下的具体脚本名
- 前置教程: TUT-0101 ~ TUT-0106

## 结构 (6 段)
1. **为什么读这个** — 你会拿到什么结果 (可交付物)
2. **前置准备** — 系统 / Python / 数据集,每条带版本号
3. **动手步骤** — 命令清单,每步可复制粘贴
4. **预期输出** — 长什么样 (截图位置 + 文字描述)
5. **常见坑** — 至少 5 条 "如果你看到 X,先检查 Y"
6. **下一步** — 链到下一节实操教程

## 字数: 1500-2500 字 (含代码)

## 必填字段
- `prerequisites`: 软硬件 + 教程依赖
- `commands`: 命令列表,每条带预期输出
- `outputExample`: 真实输出片段 (或截图路径)
- `linkedAssets`: `projectAssets.json` 中的 ID

## 代码规范
- 所有命令必须真实可执行 (作者必须自己跑过)
- 标注系统 (Linux / macOS / WSL / Windows PowerShell)
- 路径用相对路径,不用绝对路径

## 严禁
- ❌ "执行 `run.sh`" — 给出完整命令
- ❌ 跳步骤 ("然后配置好环境")
- ❌ 不给预期输出
